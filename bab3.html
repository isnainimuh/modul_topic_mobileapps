Test RESTful API

Pada bab sebelumnya, kita telah membangun RESTful API yang lengkap:

  • Products API: catalog, search, filter, categories
  • Cart API: add, update, remove, clear cart
  • Orders API: create order, tracking, status management
  • Payments API: payment processing, status update
  • Error handling dan teknik validasi yang robust

    • Tahap selanjutnya kita akan melakukan pengujian API menggunakan POSTMAN.

    TEST 1: Get Products

    
GET http://ecommerce.test/api/products

    Hasil yang diharapkan page menampilkan list semua produk dengan gambar dan kategori

    TEST 2: Get Cart (User 1)

    
GET http://ecommerce.test/api/cart/1

    Hasil yang diharapkan cart kosong atau error cart not found

    TEST 3: Add Product to Cart

    
POST http://ecommerce.test/api/cart/1/add

    Hasil yang diharapkan penambahan produk ke keranjang berhasil

    TEST 4: Get Cart Again

    
GET http://ecommerce.test/api/cart/1

    Hasil yang diharapkan menampilkan produk di cart 1

    TEST 5: Create Order

    
POST http://ecommerce.test/api/orders

    Hasil yang diharapkan membuat order dan menampilkan jumlah total

    TEST 6: Create Payment

    
POST http://ecommerce.test/api/payments

    Masukkan order_id = 1 dan method = bank_transfer

    TEST 7: Get User Orders

    
POST http://ecommerce.test/api/orders

    Masukkan user_id yang melakukan payment tadi

    TEST 8: Track Order

    
GET http://ecommerce.test/api/orders/1/track

    Sesuaikan id dengan user_id yang berhasil melakukan payment
